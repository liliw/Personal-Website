Predicting the Outcome of NBA in Python

Tool: The Jupyter Notebook, Python 3, scikit-learn, numpy, AWS glyphicon
 This project was built on Jupyter Notebook with Python 3, scikit-learn, and numpy, with 60.6% accuracy. The dataset was fetched from NBA official website via network packets. Applying with decision tree, 80% of dataset was used as training dataset; 20% dataset was used as test dataset. Before starting predicting, the pre-process included generating missing data by the column's mean value, and converting team names from string into integer by LabelEncoder.In the decision tree, there are four decision nodes:
Which team has higher ranking.
  - It is a general view.
Which team is home team
  - Home team usually has more advantages: crowd noise, comfort of being at home, etc.
Which team won their last match game.
  - They may have strategies which can against another team.
  My next step is implementing a random forest which can improve the prediction accuracy.

Travel Guides Search Engine

Tool: C/C++, PHP, SQL, TF-IDF, Cosine Similarity, Inverted Index, Information Retrieval
  It was a search engine which provided international visitors travel information of Taiwan. We wrote a PHP web crawler to collect travel information from Taiwan Tourism Bureau and a Taiwanese forum, and stored dataset as simple text. With a API which parses Mandarin sentences into elementary words, the elementary words were converted into SCV format.
  The weights of words/information were given by TF-IDF, and Cosine Similarity. User can retrieve travel information from the huge dataset which is stored in our server. However, the calculation speed is terrible due to huge data. To improve the efficiency, we implemented Inverted Index to significantly improve the search process.
